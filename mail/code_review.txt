Code Review Checklist – PHP / MySQL (OOP)

Project & structuur
☐ Mappenstructuur is logisch en overzichtelijk (bijv. app/, public/, sql/)
☐ Bestanden hebben duidelijke en consistente namen
☐ Een centraal bootstrap/config bestand wordt gebruikt
☐ Logica (PHP) is gescheiden van presentatie (HTML)
Database & ontwerp
☐ Database is aangemaakt zonder fouten
☐ Tabellen zijn correct aangemaakt
☐ Foreign keys zijn correct ingesteld
☐ Database komt overeen met het ontwerp (ERD)
☐ Seed- of testdata is aanwezig indien nodig
Security & authenticatie
☐ Wachtwoorden worden gehashed opgeslagen
☐ Inloggen gebruikt password_verify()
☐ Sessies worden correct gebruikt
☐ Beveiligde pagina’s zijn alleen toegankelijk na login
☐ Gevoelige data staat niet in URL of HTML
CRUD & functionaliteit
☐ Create (toevoegen) werkt correct
☐ Read (overzicht/bekijken) werkt correct
☐ Update (bewerken) werkt correct
☐ Delete (verwijderen) werkt correct
☐ Gebruiker ziet alleen data die hij mag zien
Dataverwerking (PDO / SQL)
☐ PDO wordt gebruikt voor databaseconnectie
☐ Prepared statements worden gebruikt
☐ Geen SQL-injectie mogelijk
☐ Queries bevatten duidelijke WHERE-clausules
Validatie & foutafhandeling
☐ Verplichte velden worden gecontroleerd
☐ Ongeldige invoer wordt afgekeurd
☐ Duidelijke foutmeldingen voor de gebruiker
☐ Geen technische foutmeldingen zichtbaar in browser
Codekwaliteit & OOP
☐ Klassen hebben één duidelijke verantwoordelijkheid
☐ Methoden hebben logische namen
☐ Code is leesbaar en consistent
☐ Geen dubbele code (DRY)
☐ Geen debug-code achtergelaten
Direct afkeuren als:
☐ Wachtwoorden in plain text worden opgeslagen
☐ SQL-injectie mogelijk is
☐ Gebruikers data van anderen kunnen zien of aanpassen
☐ Applicatie crasht bij normale invoer
